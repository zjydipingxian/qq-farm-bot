# Testing Strategy

本文档定义本项目的测试策略。当前仓库已有 lint 和 build 脚本，但测试体系仍需要逐步补齐。

## 测试目标

- 验证核心自动化逻辑不会误操作账号数据。
- 验证后端 API、认证、持久化和调度行为稳定。
- 验证前端关键页面、Pinia store 和 API 调用行为稳定。
- 避免测试访问真实 QQ 农场接口或真实通知服务。
- 把任务完成绑定到确定性证据，而不是手工判断。

## 验证协议

测试和检查按风险从小到大选择：

| 改动类型 | 最小验证 | 强验证 |
| --- | --- | --- |
| 文档 | `git diff --check` | 人工审阅链接和命令 |
| 后端类型或逻辑 | `pnpm -C core build:ts` | 后端单元测试、集成测试 |
| 前端组件或页面 | `pnpm -C web build` | 组件测试、E2E |
| 跨包接口 | `pnpm build` | 前后端集成测试 |
| 认证、账号、调度、持久化 | 对应测试 + build | 回归测试 + mock 数据验证 |

如果没有现成测试，应至少运行 build/typecheck，并在完成报告中说明测试缺口。

## 测试分层

### 单元测试

用于验证纯函数、策略判断、配置转换和 store 行为。

适合覆盖：

- 农场种植策略。
- 背包、仓库、作物收益计算。
- 好友访问策略。
- 配置读写转换。
- 前端 Pinia store。
- 前端通用组件状态。

### 集成测试

用于验证多个模块一起工作时的行为。

适合覆盖：

- Express 路由。
- 登录和 JWT 鉴权。
- 本地 JSON store 读写。
- Socket.io 事件推送。
- 后端服务层与持久化层组合。

### E2E 测试

用于验证主要用户路径。

适合覆盖：

- 登录。
- 查看仪表盘。
- 添加或编辑账号。
- 调整自动化设置。
- 查看农场、好友、分析页面。

E2E 测试应使用测试环境和假数据，不应连接真实账号。

## 后端测试约束

后端测试应放在：

```text
core/test/
core/src/**/__tests__/
```

测试数据应放在：

```text
core/test/fixtures/
```

后端测试必须 mock：

- QQ 农场网络接口。
- 定时器和调度器。
- 文件系统中的真实 `core/data/`。
- 通知服务。
- 时间相关逻辑。

后端测试应产生可读失败信息，能区分：

- 输入数据错误。
- 业务规则错误。
- 网络 mock 错误。
- 持久化错误。
- 权限错误。

不得在测试中读写：

- `core/data/accounts.json`
- `core/data/store.json`
- `core/data/users.json`
- `.env`

## 前端测试约束

前端测试应放在：

```text
web/src/**/*.test.ts
web/e2e/
```

前端测试应 mock：

- HTTP API。
- Socket.io 客户端。
- 浏览器本地存储。
- 时间和定时刷新逻辑。

组件测试应优先覆盖用户可见行为，不要只测试实现细节。

前端测试应优先验证：

- 用户能否看到正确状态。
- 用户操作是否触发正确 store 或 API。
- 加载、空状态、错误状态是否可见。
- 权限或登录状态变化是否正确跳转。

## 推荐工具

后端建议：

- Vitest：单元测试与集成测试。
- Supertest：Express API 测试。

前端建议：

- Vitest：组件和 store 测试。
- Vue Test Utils：Vue 组件测试。
- Playwright：E2E 测试。

## 必须补测试的改动

以下改动应补充或更新测试：

- 登录、注册、JWT、权限逻辑。
- 账号添加、删除、更新、批量导入。
- 自动化任务调度。
- 农场种植、收获、施肥、偷菜逻辑。
- 好友访问、黑名单、雷达逻辑。
- 本地数据读写、迁移、默认值。
- 前后端接口字段变更。

这些区域属于高风险边界。没有测试时，完成报告必须明确标记剩余风险。

## 回归复现协议

修 bug 时优先按以下顺序处理：

1. 用测试、脚本或手动步骤复现问题。
2. 记录期望行为和实际行为。
3. 归因失败类型。
4. 编写或更新能覆盖该问题的测试。
5. 修复问题。
6. 重新运行复现步骤和相关构建检查。

如果无法自动化复现，应在完成报告中写清楚手动验证步骤。

## 当前建议补齐脚本

后续可在根目录增加：

```bash
pnpm test
pnpm test:core
pnpm test:web
pnpm test:e2e
```

在 `core/package.json` 增加：

```bash
pnpm -C core test
```

在 `web/package.json` 增加：

```bash
pnpm -C web test
```

## 测试完成标准

一次涉及代码行为的改动完成前，应确认：

- 新增或变更逻辑有对应测试，或说明为什么暂不需要测试。
- 测试没有真实网络依赖。
- 测试没有读写真实运行数据。
- 相关 build 或 typecheck 已通过。
